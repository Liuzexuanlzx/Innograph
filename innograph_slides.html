Problem & Motivation
MOTIVATION

Why Citation Graphs Are Not Enough

"Paper A cites Paper B" but how does A improve upon B?
In what dimension? By what operation?

The Research Pain Point

  • Citation graphs show links, not innovations
  • Researchers spend hours manually tracing lineage
  • No tool explains what changed and why it matters

InnoGraph's Core Question

"How does this paper improve upon its predecessors — in accuracy, efficiency, or architecture?"

We build a speedy, evidence-grounded innovation graph automatically — no manual reading required.

Core Innovations

Three Unique Advantages

📊

3-Layer Innovation Taxonomy

Unlike generic "cites" edges, we classify every relationship by:

  • Type: Improves On, Extends, Combines With...
  • Dimension: Accuracy, Efficiency, Scalability...
  • Operation: Adds Module, Replaces Backbone...
Tells you exactly how one paper advanced another
✅

Independent Hallucination Verifier

By separating generation and verification:

  • Dedicated Verifier agent cross-checks every claim
  • Assigns confidence scores (0–1)
  • Rejects edges below 0.5 threshold
Reduced false claims by over 80%
Every edge backed by direct text evidence
📝

Automatic Innovation Narrative

We don't just show nodes and lines:

  • Tell the complete story of an idea's origin
  • Describe core contribution
  • Map subsequent research branches
Generates human-readable narratives automatically

No existing academic tool offers these three capabilities combined

Engineering Challenges

Key Problems We Solved

① Controlling LLM Hallucinations

Problem: Universal issue in AI applications — LLMs invent plausible-sounding but false claims.

Solution: Dedicated verification agent + strict output schema enforcement

② API Rate Limits

Problem: Multiple concurrent API calls hitting rate limits

Solution: Implemented retry logic with exponential backoff

③ Consistent Output Formats

Problem: Different LLM responses with varying structure

Solution: Custom JSON parser + explicit schema validation

④ Pipeline Speed Optimization

Problem: Sequential processing too slow for practical use

Solution: Full async concurrency → end-to-end runtime under 2 minutes

Key insight: Modular multi-agent architecture allows independent optimization of each stage
